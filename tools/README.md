# Lean Agent Framework — MCP Tools

Lightweight MCP (Model Context Protocol) servers that handle mechanical tasks so the LLM can focus on intelligence.

## Philosophy

- **Tools handle mechanics, LLM handles decisions**
- Each tool is a standalone MCP server (Python, stdio transport)
- All tools have graceful fallback — if unavailable, the LLM can do it manually
- Tools never make architectural choices

## Available Tools

| Tool | Purpose | Used By |
|------|---------|---------|
| `repo-index` | Validate and inspect child repo references in `.repo-index.yml` | Orchestrator |
| `contract-compliance` | Compare implemented routes to `.contracts/*.yml` endpoints | Orchestrator |
| `scaffold-generator` | Generate non-overwriting FastAPI/TypeScript stubs from contracts | Specialists |
| `azure-inspector` | Read Container Apps, Cosmos DB, and ACR state via Azure CLI | Specialists (infra) |
| `azure-resource-status` | Inventory Azure resources and inspect status/error events | Specialists (infra) |
| `ci-monitor` | Summarize recent GitHub Actions runs and failure hints | Orchestrator |
| `deploy-verifier` | Probe `/health`/`/version` style endpoints after deploy | Orchestrator |
| `security-scanner` | Run available scanners and normalize findings into one report | Specialists |
| `lint-local` | Run safe local lint commands (ruff/eslint/golangci-lint/shellcheck) | Specialists |
| `terraform-local` | Run deterministic local terraform fmt/init/validate/plan checks | Specialists (infra) |
| `usage-tracker` | Append usage events, summarize recent workflow activity, and report usage quality/anomalies | All agents |
| `git-pr-orchestrator` | Automate multi-repo release workflows: commit, PR creation, CI monitoring, auto-merge | Orchestrator |

## Installation

```bash
cd tools/
pip install -r requirements.txt
```

## Usage with Copilot CLI

Tools are registered in `.copilot/mcp.json` (generated by init.sh). Each tool runs as a stdio-based MCP server that Copilot CLI connects to automatically.

## Writing Custom Tools

Each tool follows the same pattern:
1. Single `server.py` file using the `mcp` Python SDK
2. Exposes one or more tools via `@server.tool()`
3. Returns structured JSON results
4. Handles errors gracefully with informative messages
